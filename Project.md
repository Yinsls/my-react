# 项目配置

> UI: ant design4.x 打包工具: webpack 路由: react-router js: typescript 状态管理: redux react-redux

```javascript
网站平台 => 查看所有内容 => 用户主页
用户主页 => 自我定制、完善，不提供用户主页，用户通过搜索昵称方式访问自己作品
作品展示: markdown => html  检测是否作者，true: 可编辑，可回复，可申请删除评论
注册只填写邮箱(唯一)、验证码、昵称、密码、确认密码
'登录'才可编辑、回复、点赞等操作，否则只可浏览
登录后一定时期内可无校验的操作，否则特殊操作需验证（邮箱码验证）

页面ui:
  首页: 最热推荐、内容列表、个人信息(设置页)            完成度(0)
  登录(注册)页:                                     完成度(85),欠缺验证码、忘记密码
  内容展示页: markdown展示内容、编辑、评论、点赞、关注    完成度(0)
  编辑器                                            完成度(0)
  设置页:
    基础设置: 头像、昵称、邮箱、密码
    统计信息: 文章数量、点赞数、关注数
```

macOS UI

```javascript
  All: 所有功能以弹框方式展示
  1、底部导航栏(程序坞)
    1.1、鼠标悬浮-小手样式(已完成)
    1.2、鼠标悬浮-图标渐渐放大，并显示程序名称(50%)
    1.3、鼠标悬浮-程序坞根据图标放大而适配图标宽度(已完成)
    1.4、程序长按选中滑动更改图标位置(本地存储程序坞位置)
    1.5、显示正开启的程序(弹窗)

    缺点: hover效果与drag有部分冲突，导致误触发hover效果
    待完善: drag图标时该元素脱离dom,导致程序坞未分配其存放空间,尤其插入和切出时由为扎眼
  2、头部导航
    2.1、当前日期时间
    2.2、搜索图标(点击显示搜索文章、昵称)
    2.3、音乐图标(用户自己上传音乐)(暂不纳入)
    2.4、最左侧logo图标(点击显示设置、关机、退出登录)
    2.5、归纳窗图标(创建归纳窗，窗体显示: 名称、关闭按钮、销毁(若其内存在软件，自动将其按其内顺序排列到桌面末尾)，可自由设定弹框大小)，桌面时显示缩略图）
    2.6、文件(导航图标或软件方式显示??，其内显示发表的文档,直接点击弹框展示html内容，长按时弹出发表、不发表、编辑、删除、重命名等操作)
    2.7、好友头像(弹窗显示)
    2.8、最右侧菜单图标(点击从右侧弹出抽屉，显示关注作者发表的文章)
  3、桌面软件图标(关注作者，流水排列)
    3.1、图标可拖动，网格放置
    3.2、可设定归纳窗
    3.3、书签功能(点击后浏览器打开此书签存储的网址)(暂不纳入)
  4、弹框
    4.1、背景毛玻璃
    4.2、标题左侧关闭、最小化、最大化，标题居中显示
    4.3、弹框四周边缘可拉缩
  5、左侧抽屉
    5.1、展示文章列表(点击后弹窗显示其内容)
    5.2、可自定义展示长宽，或是否展示(若不展示时处于隐藏状态，鼠标滑动到最左侧时弹出，点击桌面后自动隐藏)
  6、右侧抽屉
    6.1、作者栏(点击作者弹窗展示其所有文章)
    6.2、作者列表可通过长按切换顺序(排序，默认按照关注先后顺序排列)(长按显示取关、赞)
    6.3、作品栏(点击作品弹窗展示作品内容)
    6.4、作品列表按照作者栏顺序展示排序(长按显示赞、删除、下载)
```
